# Positioning Angles Context (HushAway)

**Skill:** /positioning-angles
**Gate:** Angle Gate (check-angle-gate.sh)

---

## Input

**Read:** `.claude/scratchpad/research-summary.md` (NOT full research file)

This summary is generated by `generate-research-summary.sh` and contains:
- Target keyword + secondary keywords
- Key stats for citations
- E-E-A-T citations (auto-pulled by pillar)
- Competitor gaps
- Top PAA questions

---

## Check Before Running

**Read:** `.claude/angle-library.md` to see:
- Angles already used across articles
- Patterns to avoid (don't repeat within same pillar)
- Psychology types already covered

---

## Output Requirements

Provide 3-5 angle options, each with:
- **One sentence angle** (the positioning statement)
- **Psychology/reasoning** (why this angle works)
- **Headline direction** (example headline using this angle)

---

## Session 1 Workflow (MANDATORY)

When `/positioning-angles` runs for a HushAway article in Session 1, you MUST complete these steps:

### Step 1: Generate Angles
- Read `.claude/scratchpad/research-summary.md` for article context
- Read `.claude/angle-library.md` for angles/patterns already used
- Generate 3-5 angle options with fresh patterns

### Step 2: Save ALL Angle Options to Research File
Edit the research file to add a new section after the existing content:

```markdown
---

## Generated Angle Options (Pending Selection)

**Option 1: [Angle Name]**
- Angle: [one sentence positioning]
- Psychology: [why this works]
- Headline direction: "[example headline]"
- Pattern: [new/reused] - [pattern name if reused]

**Option 2: [Angle Name]**
...

[Continue for all 3-5 options]

---
Writer selects angle at start of Session 2.
```

### Step 3: Update Research File Frontmatter
Add this field to mark angles as pending:

```yaml
angleStatus: pending-selection
```

### Step 4: Update research-summary.md
Edit `.claude/scratchpad/research-summary.md` to replace the empty Positioning section:

```markdown
## Positioning (Pending Selection)

**Status:** Angle options generated - writer selects at Session 2 start

**Option 1:** [Angle Name] - "[headline direction]"
**Option 2:** [Angle Name] - "[headline direction]"
**Option 3:** [Angle Name] - "[headline direction]"
[etc.]

Full option details in research file.
```

### Step 5: Confirm Session 1 Complete
Output confirmation:
```
SESSION 1 COMPLETE

Angle options saved to:
- Research file: [path]
- Research summary: .claude/scratchpad/research-summary.md

NEXT: Start Session 2 to select angle and write article.
```

---

## CRITICAL: Session 1 Boundaries

- **DO NOT** select an angle
- **DO NOT** update angle-library.md
- **DO NOT** run Angle Gate (it requires selection)
- **DO** save all options for Session 2 writer to review

Session 1 ends with options saved. Selection happens in Session 2.

---

## Session 2: Angle Selection (Writer Performs These Steps)

**These steps happen in SESSION 2, after the writer reviews angle options.**

1. **Select ONE angle** based on:
   - Differentiation from existing angles in library
   - Alignment with HushAway brand voice
   - Emotional resonance for target parent

2. **Update research file frontmatter:**
   ```yaml
   selectedAngle: [one sentence angle]
   headlineDirection: [headline example]
   ```

3. **Update angle-library.md** with selected angle (all 6 columns required)

4. **Run gate script:**
   ```bash
   .claude/scripts/check-angle-gate.sh [research-file]
   ```

5. **MUST show:** `ANGLE GATE: PASS`

---

## Common Failures

- Reusing an angle from same pillar
- Missing `selectedAngle` in frontmatter
- Not updating angle-library.md
